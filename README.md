# Interactive Fishplot

Interactive Fishplot is an intuitive and reproducible Shiny application to explore clonal dynamics over time.
It lets users generate fishplots, mutation trees, and quantitative matrices from structured patient data without requiring programming skills.

## Features

- Upload JSON data files
- Explore multiple visualization modes:
	- Fishplot
	- Mutation tree
	- Cumulative matrix
	- Raw data overview
- Edit data interactively:
	- Add timepoints
	- Add mutations
	- Edit mutation percentages
	- Edit drug effects
	- Add new drugs
	- Delete timepoints
	- Delete mutations
- View and edit metadata
- Built-in error handling and user feedback

## Running the App

There are two ways to run the app: locally (R/RStudio) or via the hosted ShinyApps.io version.

### Hosted Version (shinyapps.io)

https://madrigo08.shinyapps.io/interactive-fishplot/

### Run Locally

1. Clone this repository.
2. Open R or RStudio.
3. Set the working directory to the project root.
4. Install required packages if needed.
5. Run:

```r
shiny::runApp("app")
```

## Deployment

To deploy on ShinyApps.io from the project root:

```r
library(rsconnect)
setwd("C:/Users/dogom/Desktop/interactive-fishplot")
rsconnect::deployApp(
	appDir = ".",
	appPrimaryDoc = "app/app.R",
	appName = "interactive-fishplot"
)
```

## Project Structure

- app/: Shiny application source
	- app/app.R: application entry point
	- app/ui.R: UI definition
	- app/server/: modular server logic
	- app/rsconnect/: deployment metadata generated by rsconnect
- data/: example JSON data files used by the app
- functions/: data processing pipeline scripts (JSON extraction, tree building, matrices, constraints, plotting)
- fishplot_code/: custom fishplot helper code
- mutation_tree_code/: mutation tree plotting and helpers
- fishplot_manual_setup/: local fallback setup for the fishplot package (includes vendored fishplot source)
- fishplots_png_trees_aml_morita/: generated PNG outputs
- texts_and_extras/: user-facing documentation assets (guided tours, intro HTML, data format guide)
- global.R: shared libraries, helper functions, and app-wide objects
- dependencies.R: package loading/installation logic
- clones_df.csv: tabular clone data example
- metadata_df.csv: tabular metadata example
- fishplot_cache_trees_aml_morita.RData: cached R objects for faster workflows

## Usage

1. Upload a JSON file using the sidebar input.
2. Select a patient and timepoint.
3. Navigate between fishplot, mutation tree, matrix, and data views.
4. Modify percentages, mutations, timepoints, and drug effects.
5. Export modified data when needed.

## Data Format

The customized fishplot workflow is based on the Morita et al. (2020) AML dataset.
The source data contains 153 acute myeloid leukemia (AML) point-mutation trees from 123 patients (scDNA-seq, SCITE inference).

Data is stored as a nested JSON-like structure representing clonal evolution trees and sample metadata.

### Sample Naming

Each sample uses the following pattern:

AML-{PATIENT_ID}-{TIMEPOINT}

Example: AML-63-005

### Tree Structure

- Each sample corresponds to one tree
- Nodes are hierarchically linked
- Nodes can have zero or more children
- The root node represents the ancestral clone

### Node Attributes

| Attribute | Type | Description |
|---|---|---|
| node_id | string or integer | Unique node identifier |
| label | list (optional) | Usually ["Root"] |
| name | string | String representation of the node |
| matching_label | integer (optional) | Mapping identifier |
| size_percent | float (optional) | Proportion of cells |
| gene_events | object | Gene-to-mutation mapping |
